AWS Batch

Actividade práctica sobre AWS Batch

Obxectivo

O obxectivo desta actividade é comprobar de maneira práctica os conceptos básicos do servizo AWS Batch.

A xustificación da realización da práctica farase mediante a entrega na tarefa configurada na Aula Cesga dunha memoria en formato PDF que conteña as capturas de pantalla que se piden para xustificar a práctica.

Para realizar esta actividade vamos usar como base este curso dispoñíbel na plataforma AWS Workshop Studio. O curso só está dispoñíbel en inglés, pero podes axudarte, p. ex., da tradución automática integrada no teu navegador se o precisas.

Nota

Para realizar a actividade sigue as instrucións do curso tendo en conta a medida que avances as indicacións que se che dan nesta páxina.

A continuación recóllense as indicacións adicionais que terás que ter en conta en cada sección do exemplo:

Workshop setup

Para realizar a actividade vamos utilizar o Learner Lab de AWS Academy. Fai o seguinte:

  1. Accede á consola AWS desde a túa conta do Learner Lab de AWS Academy.
  2. Unha vez na consola, sigue as instrucións do curso para acceder a un terminal CloudShell e comprobar que podes usar a CLI de AWS.

Nesta sección sigue os pasos do curso.

Nota

Para subir a imaxe Docker a Amazon ECR accede ao servizo ECR desde a consola AWS do Learner Lab, non usando as ligazóns do curso.

Set up AWS Batch resources

Nesta sección sigue os pasos do curso tendo en conta o seguinte:

Nota

Accede ao servizo AWS Batch desde a consola AWS do Learner Lab, non usando as ligazóns do curso.

Set up a Compute Environment

  • No paso 6, na propiedade Instance role escolle LabInstanceProfile.

Set up a Job Definition

  • Antes de premer Next no paso 9, escolle LabRole nas propiedades Execution role e Job role configuration.

Run batch jobs

Nesta sección fai só os dous primeiros apartados (Run a single job e Run an array job) seguindo os pasos do curso.

Nota

Actualiza a información do traballos e dos seus logs a medida que se executen premendo nos botóns que refrescan a información da consola.

Optimize cost with EC2 Spot

Sáltate esta sección. O Learner Lab de AWS Academy non permite utilizar instancias EC2 de tipo spot.

Next steps and clean up

Importante

Non esquezas tomar as capturas que se piden para xustificar a práctica antes de eliminar os recursos.

Sigue os pasos do apartado Clean up tendo en conta o seguinte:

  • No paso 5, desactiva o Compute environment antes de eliminalo.
  • Non fagas os pasos 7 e 8: Empty the S3 bucket e Delete the CloudFormation stack.

Xustificación da práctica

Toma as seguintes capturas para a memoria de xustificación da práctica:

Importante

As capturas teñen que incluír o menú superior da consola AWS no que poida verse claramente teu nome de usuario. De non cumprirse este requisito a práctica non será válida.

Building a Docker container
  1. Imaxe stress-ng:latest subida ao repositorio do Amazon ECR. A captura ten que mostrar a mesma información que o exemplo do curso.
Set up AWS Batch resources
  1. Propiedades do ambiente de computación stress-ng-ec2. A captura ten que mostrar a mesma información que o exemplo do curso.
  2. Propiedades da fila de traballos stress-ng-queue. A captura ten que mostrar a mesma información que o exemplo do curso.
  3. Propiedades da definición do traballo stress-ng-job-definition. A captura ten que mostrar a mesma información que o exemplo do curso.
Run batch jobs (single job)
  1. Log da execución do traballo stress-ng-job no Cloudwatch. A captura ten que mostrar a mesma información que o exemplo do curso.

  2. Terminal do CloudShell mostrando a saída da execución do comando aws batch describe-jobs --jobs <YOUR-JOB-ID> para o traballo stress-ng-1 en estado SUCCEEDED.
