AWS Batch

Actividad práctica sobre AWS Batch

Objetivo

El objetivo de esta actividad es comprobar de manera práctica los conceptos básicos del servicio AWS Batch.

La justificación de la realización de la práctica se hará mediante la entrega en la tarea configurada en el Aula Cesga de una memoria en formato PDF que contenga las capturas de pantalla que se piden para justificar la práctica.

Para realizar esta actividad vamos a usar como base este curso disponible en la plataforma AWS Workshop Studio. El curso solo está disponible en inglés, pero puedes ayudarte, p. ej., de la traducción automática integrada en tu navegador si lo necesitas.

Nota

Para realizar la actividad sigue las instrucciones del curso teniendo en cuenta a medida que avances las indicaciones que se te dan en esta página.

A continuación se recogen las indicaciones adicionales que tendrás que tener en cuenta en cada sección del ejemplo:

Workshop setup

Para realizar la actividad vamos a utilizar el Learner Lab de AWS Academy. Sigue los pasos siguientes:

  1. Accede a la consola AWS desde tu cuenta del Learner Lab de AWS Academy.
  2. Una vez en la consola, sigue las instrucciones del curso para acceder a una terminal CloudShell y comprobar que puedes usar la CLI de AWS.

Building a Docker container

En esta sección sigue los pasos del curso.

Nota

Para subir la imagen Docker a Amazon ECR accede al servicio ECR desde la consola AWS del Learner Lab, no usando los enlaces del curso.

Set up AWS Batch resources

En esta sección sigue los pasos del curso teniendo en cuenta lo siguiente:

Nota

Accede al servicio AWS Batch desde la consola AWS del Learner Lab, no usando los enlaces del curso.

Set up a Compute Environment

  • En el paso 6, en la propiedad Instance role elige LabInstanceProfile.

Set up a Job Definition

  • Antes de pulsar Next en el paso 9, elige LabRole en las propiedades Execution role y Job role configuration.

Run batch jobs

En esta sección haz solo los dos primeros apartados (Run a single job y Run an array job) siguiendo los pasos del curso.

Nota

Actualiza la información de los trabajos y de sus logs a medida que se ejecuten presionando los botones que refrescan la información de la consola.

Optimize cost with EC2 Spot

Sáltate esta sección. El Learner Lab de AWS Academy no permite utilizar instancias EC2 de tipo spot.

Next steps and clean up

Importante

No olvides tomar las capturas que se piden para justificar la práctica antes de eliminar los recursos.

Sigue los pasos del apartado Clean up teniendo en cuenta lo siguiente:

  • En el paso 5, desactiva el Compute environment antes de eliminarlo.
  • No hagas los pasos 7 y 8: Empty the S3 bucket y Delete the CloudFormation stack.

Justificación de la práctica

Toma las siguientes capturas para la memoria de justificación de la práctica:

Importante

Las capturas tienen que incluír el menú superior de la consola AWS en el que pueda verse claramente tu nombre de usuario. De no cumplirse este requisito la práctica no será válida.

Building a Docker container
  1. Imagen stress-ng:latest subida al repositorio de Amazon ECR. Tiene que mostrar la misma información que el ejemplo del curso.
Set up AWS Batch resources
  1. Propiedades del entorno de computación stress-ng-ec2. La captura tiene que mostrar la misma información que el ejemplo del curso.
  2. Propiedades de la cola de trabajos stress-ng-queue. La captura tiene que mostrar la misma información que el ejemplo del curso.
  3. Propiedades de la definición del trabajo stress-ng-job-definition. La captura tiene que mostrar la misma información que el ejemplo del curso.
Run batch jobs (single job)
  1. Log de la ejecución del trabajo stress-ng-job en Cloudwatch. La captura tiene que mostrar la misma información que el ejemplo del curso.

  2. Terminal de CloudShell mostrando la salida de la ejecución del comando aws batch describe-jobs --jobs <YOUR-JOB-ID> para el trabajo stress-ng-1 en estado SUCCEEDED.

    Ejemplo

Run batch jobs (array job)

  1. Propiedades en la consola de AWS Batch del array de trabajos stress-ng-array en estado SUCCEEDED y con la pestaña Child jobs seleccionada.
